# Spring Cloud 微服务实战

## 第一章

微服务的 9 大特性：

1. 服务组建化
2. 按业务组织团队
3. 做”产品“的态度
4. 智能端点于哑管道
5. 去中心化治理
6. 去中心化管理数据
    > 在微服务架构中，更加强调在各服务之间进行”无事务“的调用，对应数据的一直性，
    只要求数据在最后的处理状态是一直的即可；若在过程中发现错误，通过补偿机制来进
    行处理，使得错误数据能够达到最周一致性

7. 基础设施自动化：
    1. 自动化部署
    2. 自动化测试
8. 容错设计
9. 演讲式设计
